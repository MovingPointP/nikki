## 3. ファイル・フォルダ構成

### 3-1. 日記ファイルの保存場所

`diary/` フォルダを作成して保存する。
Obsidianと連携する場合はVault内にフォルダを作成する。

```
例：~/Documents/MyVault/diary/
```

### 3-2. プロジェクト構成

```
nikki/
├── src/                    ← フロントエンド（React + TypeScript）
│   ├── components/
│   ├── store/
│   ├── hooks/
│   └── utils/
├── src-tauri/              ← バックエンド（Rust）
│   ├── src/
│   │   ├── main.rs
│   │   └── commands.rs     ← Tauriコマンド定義
│   ├── Cargo.toml
│   └── tauri.conf.json     ← パーミッション・ウィンドウ設定
├── index.html
├── vite.config.ts
└── package.json
```

### 3-3. Vaultフォルダ構成

```
MyVault/
├── diary/              ← 日記ファイル保存先
│   ├── 2026-05-03.md
│   ├── 2026-05-02.md
│   └── ...
└── rejour/             ← Rejourアプリ用フォルダ
    └── templates/      ← テンプレート保存先
        └── default.md
```

### 3-4. ファイル命名規則

- 形式：`YYYY-MM-DD.md`（例：`2026-05-03.md`）
- 1日1ファイル
- ファイル名が日付を兼ねるため、フォルダによる年月分けは行わない
- フラット構成により、10年分（約3,650件）でも問題なく動作する

### 3-5. 起動時のファイルスキャン

- アプリ起動時に `diary/` フォルダ内の全ファイル名を一度だけ読み込み、メモリに保持する
- カレンダー表示・日記存在確認はすべてメモリ上のデータを参照する
- 新規保存・削除時にメモリを更新する

## 4. Markdownファイルフォーマット

### 4-1. フロントマター

各日記ファイルの先頭にYAML形式のフロントマター（メタデータ）を記述する。

```markdown
---
tags: [仕事, 読書]
---

## 今日の出来事

（本文）
```

### 4-2. フロントマター項目

| フィールド | 内容 |
|-----------|------|
| tags | タグのリスト形式。Obsidianのタグ形式と互換性を持たせる |

> ※ `date` はファイル名から取得するため省略する。

### 4-3. Obsidian互換性

- 文字コード：UTF-8
- 改行コード：LF
- タグ形式：フロントマターの `tags: []` 形式を使用
- 本文中の `#タグ` 記法は使用しない

## 5. 機能仕様

### 5-1. 日記の作成・編集

- 新規作成時はテンプレートを自動適用する
- エディタはMarkdown記法で入力する
- エディタとプレビューを左右に並べて表示する
- 保存はボタン操作またはショートカットキー（Ctrl/Cmd + S）で行う（自動保存なし）
- 1日1ファイル。同日に作成しようとした場合は既存ファイルを開く
- 日記の削除が可能。削除前に確認ダイアログを表示する

### 5-2. カレンダー表示

- カレンダーをモーダルで表示する
- 月単位のカレンダーを表示する
- 日記が存在する日付にはマーカー（ドット等）を表示する
- 日付をクリックすると該当日の日記を開く
- 日記が存在しない日付をクリックすると新規作成画面を開く

### 5-3. タグ管理

- フロントマターの `tags` フィールドでタグを管理する
- タグはカンマ区切りで複数設定可能
- タグの追加・削除はエディタ上のフロントマター編集で行う

### 5-4. 起動時モーダル（On This Day）

#### 概要

アプリ起動時に、過去の日記を振り返るモーダルを自動表示する。

#### 表示タイミング

- アプリ起動完了後（日記ファイルのスキャンおよび当日の日記読み込みが完了したタイミング）
- 表示するタブがすべて非アクティブの場合はモーダルを表示しない

#### タブ構成

モーダル内にタブを表示し、タブを切り替えることで日記の日付を変更できる。

| タブ名 | 対象日付 | 初期選択 |
|--------|----------|---------|
| 1か月前 | 起動日から1か月前（同日） | ○ |
| 1年前 | 起動日から1年前（同日） | |
| ランダム | `dateList` からランダムに1件選択 | |

- タブ構成（種類・数）は将来的に設定から変更できる設計とする
- 初期選択タブは「1か月前」。ただしそのタブが非アクティブの場合はアクティブなタブのうち最初のものを選択する

#### タブのアクティブ／非アクティブ

- 対応する日付の日記ファイルが存在する → アクティブ（選択・閲覧可能）
- 対応する日付の日記ファイルが存在しない → 非アクティブ（選択不可・グレーアウト表示）
- ランダムタブは `dateList` が空の場合のみ非アクティブ

#### モーダルの表示内容

- タブごとに対応する日付と日記の本文プレビューを表示する
- プレビューはMarkdownレンダリング済みの表示とする（`MarkdownPreview` コンポーネントを使用）

#### モーダルのアクション

| 操作 | 動作 |
|------|------|
| 閉じる（×ボタン、モーダル外クリック） | モーダルを閉じる。当日の日記をエディタで表示した状態に戻る |
| 「この日記を開く」ボタン | モーダルを閉じ、選択中のタブの日記をエディタで開く |

---

### 5-5. テンプレート機能

- テンプレートは `rejour/templates/` フォルダに `default.md` として保存する
- テンプレートは1つのみ
- テンプレートの内容はアプリに固定（設定画面からの編集は将来対応）

#### テンプレートで使用できる変数

| 変数 | 展開後 |
|------|--------|
| `{{date}}` | 日記の日付（例：2026-05-03） |
| `{{day}}` | 曜日（例：日曜日） |

#### デフォルトテンプレート

```markdown
---
tags: []
---

# {{date}}（{{day}}）

## 気分
⭐⭐⭐⭐⭐

## 今日の出来事


## 感じたこと

```

## 6. 画面構成

### 6-1. メイン画面（3ペイン）

```
┌─────────────┬──────────────────────┬──────────────┐
│  サイドバー  │   Markdownエディタ    │  プレビュー  │
│             │                      │              │
│ ・カレンダー │  ---                 │  # タイトル  │
│             │  tags: [仕事]        │              │
│             │  ---                 │  今日は...   │
│             │                      │              │
│             │  ## 今日の出来事      │              │
│             │  今日は...            │              │
└─────────────┴──────────────────────┴──────────────┘
```


### 6-2. 起動時モーダル（On This Day）

```
┌────────────────────────────────────────────┐
│  On This Day                           ✕   │
├──────────┬──────────┬──────────────────────┤
│  1か月前  │  1年前   │  ランダム（グレー）   │
├──────────┴──────────┴──────────────────────┤
│                                            │
│  2026-05-01（金曜日）                       │
│                                            │
│  ─────── プレビュー ───────────────────    │
│  ## 今日の出来事                            │
│  今日は...                                 │
│                                            │
├────────────────────────────────────────────┤
│                      [この日記を開く] [閉じる]│
└────────────────────────────────────────────┘
```

- グレーアウトのタブは日記が存在しない（非アクティブ）

### 6-3. 設定画面

- 日記保存フォルダのパス設定（フォルダ選択ダイアログ）。初回起動時もここで設定する
- フォルダ選択は `@tauri-apps/plugin-dialog` の `open()` を使用する
- 設定値は `@tauri-apps/plugin-store` でローカルに永続化する

## 7. 非機能要件

| 項目 | 内容 |
|------|------|
| 対応OS | Windows / Mac 対応 |
| データ保存 | すべてローカルファイル。外部サーバーへの通信なし |
| Obsidian連携 | Vault内の `diary/` フォルダを参照可能。同時編集は非推奨 |
| パフォーマンス | 起動時ファイルリストスキャンは1回のみ。以降はメモリ参照 |
| 文字コード | UTF-8 / 改行コード LF |
| セキュリティ | `tauri.conf.json` でファイルシステムアクセスを最小権限に制限する |

## 8. テスト方針

### 8-1. ユニットテスト（Vitest）

- ロジック・ユーティリティ関数を対象とする
- テスト対象の例：
  - テンプレート変数の展開処理（`{{date}}`、`{{day}}`）
  - フロントマターの除去処理
  - 日付処理（ファイル名からの日付変換など）

### 8-2. E2Eテスト（WebDriver + tauri-driver）

- `tauri-driver` でTauriアプリを起動し、WebDriverプロトコルで操作する
- 実ファイルを使ってテストする（モック不使用）
- テスト対象の例：
  - 今日の日記を新規作成・保存できる
  - カレンダーから既存の日記を開ける
  - 日記を削除すると一覧から消える
  - 未保存の状態でアプリを閉じようとすると警告が出る
